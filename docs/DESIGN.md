# Oniro Item DB - 설계 문서

> GitHub Pages 기반 아이템 검색 데이터베이스 웹사이트

---

## 1. 프로젝트 개요

### 목표
`Oniro_ItemDB_1.4.4.json` (1,385개 아이템)을 기반으로 인벤 스타일의 아이템 검색/필터링 웹사이트를 구축한다.

### 기술 스택
| 구분 | 선택 | 사유 |
|------|------|------|
| 빌드 | **없음 (순수 정적 파일)** | GitHub Pages 직접 배포, 빌드 과정 불필요 |
| HTML/CSS | **Vanilla + CSS Variables** | 프레임워크 없이 경량 구현 |
| JS | **Vanilla ES6+** | 번들러 불필요, `<script type="module">` 사용 |
| 데이터 | **JSON fetch** | 런타임에 JSON 파일을 fetch하여 메모리에 로드 |
| 배포 | **GitHub Pages** | `main` 브랜치 루트에서 직접 서빙 |

### 왜 프레임워크를 쓰지 않는가
- 데이터가 1,385건으로 클라이언트 사이드 처리에 충분히 가벼움 (JSON 2.3MB)
- GitHub Pages는 정적 파일만 서빙 가능
- 빌드 스텝 없이 즉시 수정/배포 가능

## 2. 데이터 구조 분석

### 2.1 루트 데이터베이스 (7개)

| DB명 | 건수 | 용도 |
|-------|------|------|
| 아이템 옵션+스킬 DB | 1,385 | **메인 아이템 목록** |
| 무기 피해 DB | 633 | 무기 상세 피해 계산 |
| 방어구 방어력 DB | 519 | 방어구 상세 방어력 계산 |
| 옵션 통계 | 104 | 옵션 분포 참조용 |
| 스킬 통계 | 6 | 스킬 분포 참조용 |
| 타입·희귀도 통계 | 53 | 타입별 분포 참조용 |
| 범례 | 11 | 필드 설명 및 계산식 |

### 2.2 아이템 필드 구조

```
아이템ID      : number (고유키)
내부이름      : string
에디터이름    : string (영문명)
한국어이름    : string (표시명)
타입          : string (14종)
세부타입      : string (31종, 일부 아이템 없음)
희귀도        : "일반" | "전설"
표시희귀도    : "일반" | "전설" | "마법" | "희귀"
레벨          : number
요구레벨      : number
소켓수        : number
최대소켓      : number
보조배율      : number
옵션[]        : { ID, 이름, 유형, 실제값, 최소값, 최대값 }
스킬[]        : { 타입, 이름, 발동조건, 최소레벨, 최대레벨, 최소확률%, 최대확률% }
```

### 2.3 타입 분류 (카테고리)

| 대분류 | 타입 | 건수 |
|--------|------|------|
| **무기** | 무기 | 633 |
| **방어구** | 갑옷(상의), 투구, 장갑, 신발, 벨트 | 399 |
| **장신구** | 반지, 목걸이, 귀걸이 | 75 |
| **보조** | 보조무기 | 120 |
| **기타** | 문장, 기타, 코스메틱, 없음 | 158 |

### 2.4 세부타입 (무기)
검, 석궁, 활, 도끼, 양손검, 양손도끼, 양손철퇴, 양손봉, 철퇴, 봉, 장창, 단검, 너클, 수리검, 투창, 쿠나이, 완드, 지팡이, 마법도구, 두루마리, 책

## 3. 페이지 구조

### 3.1 단일 페이지 애플리케이션 (SPA)

```
index.html          ← 진입점
├── 헤더 (로고 + 타이틀)
├── 검색 영역
│   ├── 검색바 (이름 검색)
│   ├── 카테고리 탭 (전체/무기/방어구/장신구/보조/기타)
│   ├── 필터 패널
│   │   ├── 세부타입 (체크박스/태그)
│   │   ├── 희귀도 (체크박스)
│   │   ├── 레벨 범위 (슬라이더 또는 입력)
│   │   └── 옵션 필터 (드롭다운)
│   └── 정렬 옵션 (레벨, 이름, 희귀도)
├── 아이템 목록 (테이블 뷰)
│   ├── 아이템 행 (클릭 시 상세 펼침)
│   └── 페이지네이션 또는 무한스크롤
├── 아이템 상세 패널 (모달 또는 사이드 패널)
│   ├── 기본 정보
│   ├── 옵션 목록
│   ├── 스킬 목록
│   └── 무기/방어구 상세 스탯 (서브DB 연동)
└── 푸터
```

## 5. 핵심 기능 상세

### 5.1 검색

| 기능 | 설명 |
|------|------|
| **텍스트 검색** | 한국어이름, 에디터이름 동시 검색 (대소문자 무시) |
| **실시간 필터** | 입력 시 300ms 디바운스로 즉시 결과 반영 |
| **URL 해시** | 검색 조건을 URL 해시에 저장 (`#q=활&type=무기&rarity=전설`) |
| **검색 기록** | localStorage에 최근 검색어 5개 저장 |

### 5.2 카테고리 필터

대분류 탭으로 빠른 필터링:
```
전체(1385) | 무기(633) | 방어구(399) | 장신구(75) | 보조(120) | 기타(158)
```

- 탭 선택 시 해당 카테고리만 표시
- 탭 아래에 세부타입 태그가 동적으로 표시됨

### 5.3 상세 필터

| 필터 | 타입 | 값 |
|------|------|----|
| 세부타입 | 체크박스 그룹 | 카테고리에 따라 동적 변경 |
| 표시희귀도 | 체크박스 | 일반, 마법, 희귀, 전설 |
| 레벨 범위 | 숫자 입력 2개 | min ~ max |
| 옵션 보유 | 드롭다운 | 특정 옵션을 가진 아이템만 |

### 5.4 정렬

| 기준 | 기본방향 |
|------|----------|
| 레벨 | 오름차순 (기본) |
| 이름 | 가나다순 |
| 희귀도 | 전설→희귀→마법→일반 |
| 아이템ID | 오름차순 |

### 5.5 페이지네이션
- 페이지당 30개 표시 (설정 가능: 30/50/100)
- 상단/하단 페이지 이동 버튼

### 5.6 아이템 상세 연동

아이템 클릭 시 모달에 표시하는 데이터:

1. **기본 정보**: 메인 DB에서 가져옴
2. **무기 상세**: `무기 피해 DB`에서 아이템ID로 조인
3. **방어구 상세**: `방어구 방어력 DB`에서 아이템ID로 조인
4. **옵션 한글화**: `범례` DB의 이름 매핑 활용

## 7. 성능 고려사항

|------|------|
| **초기 로딩** | JSON 파일 1회 fetch 후 메모리 캐시 |

## 9. URL 해시 스키마

```
#q=검색어&cat=무기&sub=활,검&rarity=전설&lvMin=10&lvMax=50&sort=level&order=asc&page=1
```

| 파라미터 | 설명 | 기본값 |
|----------|------|--------|
| q | 검색어 | (빈값) |
| cat | 대분류 카테고리 | all |
| sub | 세부타입 (콤마 구분) | (전체) |
| rarity | 표시희귀도 (콤마 구분) | (전체) |
| lvMin | 최소 레벨 | 1 |
| lvMax | 최대 레벨 | 999 |
| sort | 정렬 기준 | level |
| order | 정렬 방향 | asc |
| page | 현재 페이지 | 1 |
| item | 상세 보기 아이템ID | (없음) |

## 부록: 참고 사이트 디자인 패턴 (인벤 DB 스타일)

인벤 게임 DB 사이트의 일반적 패턴:
- **상단 고정 검색바**: 항상 접근 가능한 검색 입력
- **좌측 카테고리 또는 상단 탭**: 대분류 빠른 전환
- **테이블 기반 목록**: 컬럼 헤더 클릭으로 정렬
